API

L'API sert de pont de communication entre deux logiciels.
Elle permet à une application de « demander » des données à une autre application, sans avoir besoin de savoir comment cette dernière est constituée à l'intérieur.
​L'API est utilisée principalement
Pour gagner du temps : Au lieu de recréer une carte du monde entière, un développeur utilise l'API de Google Maps. ​
La sécurité : l'API permet d'accéder à des données sans donner les clés de toute la base de données. ​
L'interconnexion : quand tu veux que ton application communique avec le monde extérieur (météo, paiements, réseaux sociaux...). ​
Le processus suit généralement quatre étapes simples : ​
La requête : ton application envoie une demande. ​
La réception : l'API reçoit la demande et vérifie si tu as l'autorisation. ​
Le traitement : elle va chercher l'info dans le système distant. ​
La réponse : elle te renvoie l'information. ​
Des exemples concrets d'applications :
Wave qui est célèbre en Côte d'Ivoire, mais sa vraie force vient de son API de paiement que les autres sites utilisent. ​ Ça fonctionne comme suit : ​
La requête : le site de tickets envoie une requête à l'API de Wave. ​
L'action : l'API de Wave répond en affichant le QR code ou en ouvrant ton application Wave. ​
La confirmation : une fois que tu as validé sur ton téléphone, l'API de Wave renvoie un signal instantané au site de tickets. ​
Le résultat : le site te délivre ton ticket automatiquement sans intervention humaine.
